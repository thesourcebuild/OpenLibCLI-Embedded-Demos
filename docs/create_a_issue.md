# How to Create an Issue

Use GitHub Issues to report bugs, request features, or point out documentation problems.

## Table of Contents

- [Before You Open an Issue](#before-you-open-an-issue)
- [What to Include](#what-to-include)
- [Good Issue Reports](#good-issue-reports)
- [Where to Submit](#where-to-submit)

## Before You Open an Issue

- Search existing issues first.
- Check the latest `README.md` and `Docs/` files.
- Try to reproduce the problem with the smallest possible example.

## What to Include

- A short, clear title.
- What you expected to happen.
- What actually happened.
- Exact steps to reproduce the issue.
- **Your platform** and **toolchain version**.
- Relevant logs, compiler output, or screenshots.

## Good Issue Reports

- Keep the report focused on one problem.
- Include code snippets only when they help reproduce the issue.
- If the issue is platform-specific, mention the target board, OS, and build system.

## Where to Submit

Open a new issue on the OpenLibCLI-Embedded-Demos GitHub repository:

https://github.com/thesourcebuild/OpenLibCLI-Embedded-Demos/issues
